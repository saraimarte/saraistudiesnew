<RuleBox>
**Def.** A **network** is a system of links that interconnects nodes in order to move information between nodes.

(From a discrete 2 perspective, it's just a graph with nodes and edges.)
</RuleBox>

There are different types of networks:

<ul>
    <li>**Social Media Networks** (Nodes: People, Edges: Friendship Relationship)</li>
    <li>**Cellular Networks** (Nodes: Cell towers, Edges: Wireless Communication Links)</li>
    <li>**Transportation Networks** (Nodes: Intersections, Edges: Roads/Streets)</li>
    <li>**Academic Citation Networks** (Nodes: Research papers, Edges: Citations)</li>
    <li>**The World Wide Web** (Nodes: Webpages, Edges: Hyperlinks)</li>
    <li>**Internet** (Nodes: Computers, Edges: Network Cables)</li>
</ul>

In this class, we focus only on the Inter*net*.

<RuleBox align = "left">
**Def.** An **internet** is a generic term for any network of networks. 

Connects a specific set of devices or smaller networks.

(From a discrete 2 perspective, it's just a graph of graphs.)

</RuleBox>

<RuleBox align = "left">
**Def.** The **Internet** is a global internet managed by a language that's based on the IP protocol. 

Connects millions of private and public networks worldwide. 
</RuleBox>

<RuleBox align = "left">
**Def.** The **IP Protocol** is a set of rules allowing entirely different computer networks to communicate with one another.
</RuleBox>

<RuleBox align = "left">
**Def.** **Internet Technology** refers to the architecture, services and protocols needed for an internet to work. (or the Internet?)
</RuleBox>

The Internet started as a research experiment called **ARPANET** in the 1960s that would link university and military researchers to test a new way of communication.  

# Characteristics of the Internet 

**1. <Pop t = "the ability of different systems, devices, or applications to connect, communicate, and exchange data securely and effectively.">Interoperability</Pop> is the Internet's most important goal so to enforce interoperability, the Internet is a <Pop t = "a type of architecture where different independent entities work together as a single unit.">Federated System</Pop>**

<ul>
    <li> where the different independent entities are <Pop t = "entities that provide internet service (like Optimum, Verizon, etc.)"> Internet Service Providers.</Pop></li>
</ul>

**2. The Internet is diverse and dynamic**

There are different types of...

<ul>
    <li><Pop t = "the time it takes for data to be received">Communication latency</Pop> (microseconds to seconds.)</li>
    
    <li><Pop t = "maximum amount of data that can flow through the network connection/link/edge per second">Bandwidth</Pop> (1Kbits/second to 100 Gigabits/second)</li>
    
    <li><Pop t = "the % of data packets that fail to reach their destination">Packet loss</Pop> (0-90%)</li>    
    
    <li><Pop t = "the material in which network edges/links are made of">Technology</Pop> (optical, wireless, satellite, copper.)</li>
   
    <li><Pop t = "the physical hardware connected to the network edge (so a network node)">Endpoint Devices</Pop> (sensors cell phones, datacenters, supercomputers.)</li>    
   
    <li>Applications (social networking, file transfer, skype, live TV, gaming, remote medicine)</li>
   
    <li>Users (the governing, governed, operators, malicious, addicted, gamers, etc.)</li>
</ul>

**3. The Internet is prone to failure**

<ul>
    <li>To send data through a network, *all* components along a path must work correctly (software, firewall, links, endpoint devices, human operators etc.) </li>
    <li>*Any* part of the Internet might fail.</li>
</ul>

# What are Network Applications?

<RuleBox align = "left">
**Def.** The **Network Applications** are applications that run on different end systems in a network (e.g. Canvas, Zoom, Bloomberg, Web Servers.)
</RuleBox>

## Application Considerations 

When designing an application there are two main events to take into account. First, the input from the application to the network, and second, the output from the network back to the application. 

Within these events, there are factors to consider. 

**1. Input from the application to the network**

<ul>
    <li><Pop t = "how much data can flow over the network per unit of time">Traffic data rate</Pop></li>
    <li><Pop t = "the fluctuation of data flows">Traffic pattern</Pop> (bursty flow (text messages) or fixed/steady flow (Netflix Video Streaming))</li>
    <li><Pop t = "the intended destination of the network data">Traffic Target</Pop>(<Pop t = "multiple destinations">multipoint</Pop> or single destination, mobile or fixed)</li>
</ul>

**2. Output from the network back to the application.**

<ul>
  <li><Pop t = "how tolerant an application is to data being held up in transit.">Delay Sensitivity</Pop></li>
  <li><Pop t = "how affected the application is by missing data">Loss Sensitivity</Pop></li>
  <li><Pop t = "if device or user is moving around">Mobility,</Pop><Pop t = "where the sender or receiver are">location,</Pop> <Pop t = "if data is being sent or received by multiple endpoints">multicast</Pop> sensitivity</li>
</ul>
