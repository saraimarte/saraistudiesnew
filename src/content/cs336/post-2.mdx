# Possible Errors You May Encounter 

**1.** Build Path Error 

<ImageMd img = {img1} alt = ""/>

The following solution is from Gemni. 

<Quote author = "Gemni" source = "">
The import statement ```<%@ page import="java.sql.*" %>``` is correct. The problem is that your project is missing the main Java library.

Here is how to fix it:

    In the "Project Explorer," right-click your cs336 project.

    Go to Properties (at the bottom).

    On the left, select "Java Build Path".

    Click the "Libraries" tab at the top.

    Look at the list. You should see "JRE System Library" or "JavaSE". If it's missing or has a red 'X' on it, that's the problem.

    Click the "Add Library..." button on the right.

    Select "JRE System Library" and click "Next".

    Make sure "Workspace default JRE" is selected (or pick your "JavaSE-21" from the "Execution environment" list).

    Click "Finish".

    Click "Apply and Close".

This will add the main Java library back to your project, and Eclipse will now be able to find the Connection class. 

[Or at least it should- the error still might be there for a bit. Try cleaning the project (Project -> Clean...) and restarting Eclipse if it doesn't go away right away. That's what worked for me - Sarai. Either way, this error doesn't actually prevent anything from working.]

</Quote>
