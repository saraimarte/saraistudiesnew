<ul>
  <li>**Note on /:** In front of special characters the backslash escapes them so they become literal (looses their special meaning): `\`. means a real dot, `\$` means a real dollar sign. But in front of letters the backslash converts them into special characters: `\d` (means any digit), `\w` (means any word character), `\s` (means any whitespace character). So the same symbol either removes special meaning or adds it depending on what follows.</li>
  <li>**Note on ^:** Inside brackets, as the first symbol, means NOT so [^abc] means any character except/NOT a, b, or c </li>
  <li>**Note on -:** is a symbol that only has special meaning inside brackets. It means 'through' so [a-z] means ["a through z"] </li>
  <li>**Note on []:** All special characters (like \s, \d, etc.) lose their meaning inside brackets except ^, -, \, and ]. </li>
  <li>**Note on Groups () vs (:?):** Capture groups allow you to deconstruct a text into groups that you can work with. You will see the use of this when you get to `re.search()`. You use non-capture groups (:?) when you want to group tokens together but don't want to remember those tokens together later and just use it as a tool. </li>
</ul>


## Regex Functions

Once we are comfortable describing patterns with regex, we can pass these patterns into Python's `re` functions to search, extract, replace, or split text based on those patterns.

  <thead>
    <tr>

      <th style="padding: 12px; border-bottom: 2px solid #cbd5e1; text-align: left;">Function</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td style="padding: 12px; border-bottom: 1px solid #e2e8f0; font-weight: bold; width: 60%;">Check if a pattern exists anywhere in the text</td>

    </tr>
    <tr>
      <td style="padding: 12px; border-bottom: 1px solid #e2e8f0; font-weight: bold;">Return every substring that matches the pattern</td>

    </tr>
    <tr>
      <td style="padding: 12px; border-bottom: 1px solid #e2e8f0; font-weight: bold;">Replace every matching substring with something else</td>

    </tr>
