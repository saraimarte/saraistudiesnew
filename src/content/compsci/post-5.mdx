## Math Library

### Introduction

Have you ever asked yourself how to square a number in Java?

Well, you can multiply it by itself...

<Code lang="java" fileName="SquareNumber" code={`System.out.println(4*4); //same thing as 4^2 = 16`} />

but what if you want to cube a number?

Well, you can multiply it by itself 3 times...

<Code lang="java" fileName="CubeNumber" code={`System.out.println(4*4*4); //same thing as 4^3 = 64`} />

But what if you want to raise a number to the 12th power?

Will you multiply that number by itself 12 times?

<Code lang="java" fileName="Power12" code={`System.out.println(4 * 4 * 4 * 4 * 4 * 4 * 4 * 4 * 4 * 4 * 4 * 4); //same thing as 4^12 = 16777216`} />

...yeah no. We don't have all day.

That's why the Math library exists!

The library comes with a bunch of pre-built functions/methods you can use like the `pow` function (stands for power).

So, using the power function from the Math library...

<Code lang="java" fileName="MathPow" code={`System.out.println(Math.pow(4, 12)); //same thing as 4^12 = 16777216 but much nicer :)`} />

## Math.pow()

Raises the first input to the power of the second input.

Works with Integers

<Code lang="java" fileName="MathPowInteger" code={`System.out.println(Math.pow(5, 2)); //5^2 = 25.0`} />

Works with Negative Integers

<Code lang="java" fileName="MathPowNegativeInteger" code={`System.out.println(Math.pow(-5, 3)); //-5^3 = 125.0`} />

Works with Doubles

<Code lang="java" fileName="MathPowDouble" code={`System.out.println(Math.pow(2.4, 2)); //5.76`} />

Note: The `Math.pow` method returns only doubles no matter what you put in.

## Math.min()

Finds the lowest value out of the two inputs.

Works with Integers

<Code lang="java" fileName="MathMinInteger" code={`System.out.println(Math.min(1, 2)); //1`} />

Works with Doubles

<Code lang="java" fileName="MathMinDouble" code={`System.out.println(Math.min(1.2, 2.4)); //1.2`} />

## Math.max()

Finds the greatest value out of the two inputs.

Works with Integers

<Code lang="java" fileName="MathMaxInteger" code={`System.out.println(Math.max(1, 2)); //2`} />

Works with Doubles

<Code lang="java" fileName="MathMaxDouble" code={`System.out.println(Math.max(1.2, 2.4)); //2.4`} />

## Math.abs()

Takes the absolute value of the input.

Works with Integers

<Code lang="java" fileName="MathAbsInteger" code={`System.out.println(Math.abs(-1)); //1`} />

Works with Doubles

<Code lang="java" fileName="MathAbsDouble" code={`System.out.println(Math.abs(-1.2)); //1.2`} />

## Math.sqrt()

Squares the input.

Works with Integers

<Code lang="java" fileName="MathSqrtInteger" code={`System.out.println(Math.sqrt(4)); //2.0`} />

Does not work with negative Integers

<Code lang="java" fileName="MathSqrtNegativeInteger" code={`System.out.println(Math.sqrt(-4)); //NaN`} />

Works with Doubles

<Code lang="java" fileName="MathSqrtDouble" code={`System.out.println(Math.sqrt(5.76)); //2.4`} />

Note: Always returns doubles no matter what you put in.

## Math.ceil()

Rounds whatever you give it to the next whole number (it just rounds up).

20.9 -> 21

20.2 -> 21

21 -> 21 (so, there is really no point in providing ints as input for ceil since it will give you the same thing you gave it.)

Note: Always returns doubles no matter what you put in.

Works with Integers (no point in this)

<Code lang="java" fileName="MathCeilInteger" code={`System.out.println(Math.ceil(21)); //21`} />

Works with Doubles Ex1

<Code lang="java" fileName="MathCeilDouble1" code={`System.out.println(Math.ceil(20.9)); //21`} />

Works with Doubles Ex2

<Code lang="java" fileName="MathCeilDouble2" code={`System.out.println(Math.ceil(20.2)); //21`} />

## Math.floor()

Opposite of ceil(). This rounds down - to the FLOOR. Down. Floor. See the connection?

Note: Always returns doubles no matter what you put in.

Works with Integers (no point in this)

<Code lang="java" fileName="MathFloorInteger" code={`System.out.println(Math.floor(20)); //20`} />

Works with Doubles Ex1

<Code lang="java" fileName="MathFloorDouble1" code={`System.out.println(Math.floor(20.2)); //20.0`} />

Works with Doubles Ex2

<Code lang="java" fileName="MathFloorDouble2" code={`System.out.println(Math.floor(20.9)); //20.0`} />

## Math.PI

`Math.PI` gives you π

<Code lang="java" fileName="MathPI" code={`System.out.println(Math.PI); //3.141592653589793`} />

### Math.random()

About:

- (1) Generates a random **double** from 0 to 1.
- (2) It always includes the first number.
- (3) It does not include the last number unless you add a +1.
- (4) If you want it to be a number greater than 0 to 1 then just multiply by the last number.
- (5) Inclusive = number should be included. Exclusive = number should be excluded.

### Example 1 | Generate a random _integer_ between 0 and 1 not including 1

<Code lang="java" fileName="MathRandomInteger0To1" code={`int randomNum = (int) Math.random();
System.out.println(randomNum);`} />

### Example 2 | Generate random _integer_ between 1 and 10 (both inclusive)

<Code lang="java" fileName="MathRandomInteger1To10" code={`int randomNum = (int) (Math.random() * 10 + 1);
// OR int randomNum = (int) (Math.random() * 11);

System.out.println(randomNum);`} />

### Example 3 | Generate random _integer_ between 0 and 5 (not including 5)

<Code lang="java" fileName="MathRandomInteger0To5" code={`int randomNum = (int) (Math.random() * 5);
System.out.println(randomNum);`} />

## Conclusion

Only `Math.pow()`, `Math.sqrt()`, `Math.ceil()` and `Math.floor()` return only doubles. As in no matter what you put in - in an integer or double - you will _only_ get a double back.
